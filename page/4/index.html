It can be installed elsewhere, but that‚Äôs not how we roll here. https://linuxcontainers.org/ LXD containers are not VM‚Äôs, but are designed to work exactly like them. The key difference is they are fully native, and they access to hardware directly (well networking aside). You can even grant access to a GPU. Containers are a very flexible tool for your everyday Linux use. Ubuntu 16.04 ships with LXD 2.0, but for some of the advanced features, you‚Äôre going to want the latest. https://launchpad.net/~ubuntu-lxc/+archive/ubuntu/lxd-stable Working with containers is A LOT like working with vagrant boxes. Reference: https://github.com/lxc/lxd/issues/3333 Modern Kernels on LTS Ubuntu Starting with Ubuntu 16.04 LTS, you are able to make your Ubuntu install subscribe to the latest changes to the Linux kernel. There are 3 channels you can subscribe to: GA-16.04 (General Availability) HWE-16.04 (Hardware Enablement) HWE-16.04-Edge (Cutting Edge Hardware Enablement) By default Ubuntu puts you on the GA track, meaning in Ubuntu 16.04‚Äôs case, you‚Äôre getting Kernel 4.4.x. Switching to HWE, you get a current Kernel. At the time of this writing, that‚Äôs 4.10.x. HWE channels are good up until the next major LTS release of Ubuntu. Then you effectively get put on the GA track of the now current LTS release (i.e. 18.04 starting April 2018). It is then expected you‚Äôll upgrade to the new LTS release, where you can begin again, switching to the next HWE series.
